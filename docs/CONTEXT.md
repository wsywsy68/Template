# CONTEXT

本仓库是一个标准 Python 项目模板，专为大模型（LLM）代码生成与规范提供结构骨架。

## 设计理念

遵循"大道至简"原则，提供轻量且开箱即用的干净骨架，降低新建项目的初始化成本，建立统一的开发规范。

## 目录约定

| 目录 / 文件 | 用途 |
|---|---|
| `src/` | 核心业务逻辑代码 |
| `src/utils/` | 可复用的纯函数工具 |
| `tests/` | 单元测试 |
| `scripts/` | 一次性或辅助脚本 |
| `data/` | 原始输入数据 |
| `results/` | 程序输出结果 |
| `docs/` | 文档与上下文说明 |

## 使用建议

- 克隆本仓库作为新项目的起点，替换 `src/main.py` 中的业务逻辑即可。
- 第三方依赖统一写入 `requirements.txt`。
- 敏感配置通过环境变量注入，禁止硬编码。
- 所有 Python 代码遵循 PEP 8，函数命名使用 `snake_case`，并添加类型注解（Type Hints）。
